Ivren Service telepítési útmutató

Célközönség: üzemeltetés · Környezet: WOVER szerver · Kiadás dátuma: 2026-05-25

Röviden: az Ivren Service a bejövő PDF számlákat figyeli, megpróbálja kiolvasni belőlük a számlaszámot, majd siker esetén átnevezi és a sikeres mappába mozgatja őket. Sikertelen feldolgozás esetén a PDF a hibás mappába kerül. Minden éles feldolgozásról audit log készül.

1. Átadott telepítési csomag

Az üzemeltetés részére átadott ZIP fájl neve:

Ivren.Service-win-x64-2026-05-25.zip

A ZIP tartalma:

  • Ivren.Service.exe és a hozzá tartozó futtatási fájlok.
  • Ivren.Service.settings.json konfigurációs fájl.
  • Ivren.SupplierProfiles.json szállítói profil konfiguráció.
  • IvrenServiceTelepitesiUtmutato.pdf jelen telepítési útmutató.

2. Javasolt éles szerver mappák

FunkcióJavasolt útvonal
Service program mappájaC:\var\ivren\service
Input mappaD:\DATA\ivren\input
Sikeresen átnevezett PDF-ekD:\DATA\ivren\renamed
Sikertelen PDF-ekD:\DATA\ivren\failed
Service log és audit logD:\DATA\ivren\logs

Ajánlott service account:

MEGA\svc_ivren

3. Előfeltételek

  • Windows Server a WOVER gépen.
  • Helyi admin jogosultság a telepítés idejére.
  • A MEGA\svc_ivren service account előzetesen létrehozva, ismert jelszóval.
  • A service account kapjon Modify jogosultságot a D:\DATA\ivren mappára.
  • A service account kapjon Read & Execute jogosultságot a C:\var\ivren\service mappára.
  • A service account kapja meg a Log on as a service helyi biztonsági jogot.

4. Telepítési lépések

4.1. Mappák létrehozása

Emelt jogosultságú PowerShell ablakban futtassa:

New-Item -ItemType Directory -Force -Path `
  "C:\var\ivren\service", `
  "D:\DATA\ivren\input", `
  "D:\DATA\ivren\renamed", `
  "D:\DATA\ivren\failed", `
  "D:\DATA\ivren\logs"

4.2. ZIP kicsomagolása

Csomagolja ki a ZIP teljes tartalmát ebbe a mappába:

C:\var\ivren\service

Ellenőrizze, hogy létezik:

C:\var\ivren\service\Ivren.Service.exe
C:\var\ivren\service\Ivren.Service.settings.json
C:\var\ivren\service\Ivren.SupplierProfiles.json

4.3. Jogosultságok beállítása

icacls "D:\DATA\ivren" /grant "MEGA\svc_ivren:(OI)(CI)M" /T
icacls "C:\var\ivren\service" /grant "MEGA\svc_ivren:(OI)(CI)RX" /T

A Log on as a service jog beállítható a Local Security Policy felületen:

  1. Futtatás: secpol.msc
  2. Local Policies → User Rights Assignment
  3. Log on as a service
  4. Adja hozzá: MEGA\svc_ivren

4.4. Konfiguráció ellenőrzése

Nyissa meg:

C:\var\ivren\service\Ivren.Service.settings.json

Alapértelmezett éles konfiguráció:

{
  "InputFolder": "D:\\DATA\\ivren\\input",
  "RenamedFolder": "D:\\DATA\\ivren\\renamed",
  "FailedFolder": "D:\\DATA\\ivren\\failed",
  "AuditLogFolder": "D:\\DATA\\ivren\\logs",
  "PollIntervalSeconds": 30,
  "FileReadyDelaySeconds": 10,
  "DryRun": false
}
Első éles próba előtt ajánlott: állítsa a DryRun értéket true-ra, indítsa el a service-t, dobjon be egy teszt PDF-et, ellenőrizze a logokat, majd sikeres ellenőrzés után állítsa vissza false-ra és indítsa újra a service-t.

4.5. Windows Service létrehozása

Emelt jogosultságú parancssorból:

sc.exe create IvrenService binPath= "C:\var\ivren\service\Ivren.Service.exe" start= auto obj= "MEGA\svc_ivren" DisplayName= "Ivren PDF Invoice Rename Service"

A service account jelszavát a Windows service konfigurációban kell megadni. Ha a fenti parancs után szükséges, használható a Services MMC:

services.msc

Keresse meg: Ivren PDF Invoice Rename Service, majd Log On fülön ellenőrizze a service accountot és jelszót.

4.6. Service indítása

sc.exe start IvrenService
sc.exe query IvrenService

Sikeres állapot:

STATE : 4 RUNNING

5. Első működési ellenőrzés

  1. Másoljon egy teszt PDF-et az input mappába: D:\DATA\ivren\input.
  2. Várjon legalább FileReadyDelaySeconds + PollIntervalSeconds ideig.
  3. Siker esetén a PDF megjelenik: D:\DATA\ivren\renamed.
  4. Sikertelen feldolgozás esetén a PDF megjelenik: D:\DATA\ivren\failed.
  5. Ellenőrizze a service logot és az audit logot.

Service log

Get-Content "D:\DATA\ivren\logs\ivren-service-$(Get-Date -Format yyyy-MM-dd).log" -Tail 80

Audit log

Get-Content "D:\DATA\ivren\logs\ivren-audit-$(Get-Date -Format yyyy-MM-dd).log" -Tail 20

A service logban induláskor látszódnia kell:

Ivren PDF Invoice Rename Service starting.
Running as Windows user: MEGA\svc_ivren
Settings file: C:\var\ivren\service\Ivren.Service.settings.json
Supplier profiles file: C:\var\ivren\service\Ivren.SupplierProfiles.json

6. Frissítés új verzióra

  1. Állítsa le a service-t: sc.exe stop IvrenService.
  2. Készítsen mentést a jelenlegi C:\var\ivren\service mappáról.
  3. Csomagolja ki az új ZIP tartalmát a service mappába.
  4. Ellenőrizze, hogy az Ivren.Service.settings.json környezetspecifikus értékei megfelelőek.
  5. Indítsa el a service-t: sc.exe start IvrenService.
  6. Ellenőrizze a service logot és egy teszt PDF feldolgozását.

7. Rollback

  1. Állítsa le a service-t.
  2. Állítsa vissza a korábbi service mappa mentését.
  3. Indítsa újra a service-t.
  4. Ellenőrizze a logot és a feldolgozást.

8. Hibakeresés

JelenségTeendő
A service nem indulEllenőrizze a service account jelszavát és a Log on as a service jogot.
Nincs feldolgozásEllenőrizze, hogy az input mappa létezik-e, és közvetlenül abban vannak-e a PDF-ek.
A fájlok nem mozdulnakEllenőrizze a service logban, hogy a fájl nem túl friss vagy zárolt-e.
Minden ciklus kimaradEllenőrizze a konfigurált mappák meglétét és jogosultságait.
XML profilok nem érvényesülnekEllenőrizze, hogy az Ivren.SupplierProfiles.json az exe mellett van-e.
Nincs audit logÉles módban ellenőrizze az audit log mappa jogosultságát; dry-run módban audit log nem íródik.

9. Eltávolítás

sc.exe stop IvrenService
sc.exe delete IvrenService

Az adat- és logmappákat csak üzleti jóváhagyással szabad törölni.
